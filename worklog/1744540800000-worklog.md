# 工作日志 - 电影目录扫描功能优化

## 目标

对 `MovieService.importScannedMovies` 进行重构，优化电影目录扫描功能：

1. **目录扫描**：扫描指定目录下所有文件夹（含子文件夹），提取 `movie.nfo` 和 `*-poster.jpg` 文件，在电影库目录创建以电影ID命名的文件夹，将NFO和poster拷贝到文件夹中，同时修改源NFO文件的 `original_filename`

2. **文件扫描**：支持CSV格式导入，CSV列依次包括：电影ID、电影名称、电影描述、排序标题、演员（|分割）、导演、上映时间、发行商、电影时长(分钟)、标签、文件地址、视频编码、视频宽度、视频高度、视频时间

3. **扫描结果展示**：在 `scan-result-modal` 中通过列表展示电影ID、电影名称、演员、导演、发行商、电影时长、电影海报地址、视频文件地址

4. **编辑模态窗**：在 `scan-movie-edit-modal` 中增加电影ID、演员、导演输入框

5. **索引重建**：扫描保存完成后重新建立 `index.json`

## 工作过程

### 第一阶段：代码分析

1. **阅读现有代码结构**：
   - `MovieService.scanMovieDirectory` (line 723-798) - 当前实现只扫描子文件夹名称，未提取NFO内容
   - `MovieService.importScannedMovies` (line 856-918) - 使用moveDir移动目录
   - `FileService` - 提供文件操作基础服务

2. **理解数据流**：
   - 扫描目录 → 创建临时目录 → 用户编辑 → 导入到正式目录 → 更新缓存和索引

3. **理解NFO格式**：
   ```xml
   <movie>
       <id>movie-星际穿越-2014</id>
       <title>星际穿越</title>
       <year>2014</year>
       <director>克里斯托弗·诺兰</director>
       <original_filename>D:/Movies/星际穿越/星际穿越.mp4</original_filename>
       ...
   </movie>
   ```

### 第二阶段：实现

#### 1. FileService 新增方法

**文件**: `src/main/services/FileService.js`

新增方法：
